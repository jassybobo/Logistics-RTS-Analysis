# Logistics-RTS-Analysis
## Overview
This project analyzes Return to Sender (RTS) shipments to identify key drivers of delivery failures and reduce logistics costs.

## Business Problem
High RTS volume was increasing operational costs and reducing delivery efficiency. There was no centralized visibility into why shipments were failing.

## Objective 
The main objective of this analysis is to identify root causes of failed deliveries, reduce RTS volume, and improve operational efficiency.

## Tools Used
   - Python
   - SQL
   - Tableau/Power BI/Qlik
   - Microsoft Excel/ Pivot Table

## Procedures
### Data Extraction
   - Used SQL to pull shipment, returns, and vendor data from multiple sources.
### Data Cleaning
   - Standardized delivery status, location fields, and removed inconsistencies.
### Analysis
   - Performed exploratory analysis to identify trends in failed deliveries.
### Visualization
   - Built dashboards to track RTS trends, cost impact, and carrier performance.

### Results
   - Reduced RTS volume by 25 percent.
   - Improved decision making through better data visibility.
     
## Key Insights
  - Certain access points had significantly higher failure rates.
  - Address quality issues were a major contributor for some access points.
